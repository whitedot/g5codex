# Manual Test Scenarios

기준일: 2026-04-27

## 목적

자동 검사로 잡기 어려운 운영 흐름을 변경 작업 후 사람이 확인할 수 있게 고정한다.

## 공통 기준

- 시작 전 `git status --short`로 변경 범위를 확인한다.
- 수동 확인 전 `npm run check:refactor`, 필요한 CSS 빌드, PHP 구문 검사를 먼저 통과시킨다.
- 운영 데이터로 실행할 때는 다운로드 파일, 메일 발송, 회원 상태 변경이 실제로 발생하는지 먼저 확인한다.

## 관리자 회원 Export

1. Export 화면 진입: `adm/member_list_exel.php`가 오류 없이 열리고 총건수, 필터, 안내문, 다운로드 버튼이 표시된다.
2. 런타임 준비 실패: ZIP/XLSX 생성 환경이 준비되지 않은 경우 다운로드 버튼이 비활성화되고 안내 메시지가 표시된다.
3. 단일 XLSX 다운로드: 작은 조건으로 다운로드를 실행했을 때 progress가 완료 상태가 되고 생성 파일을 열 수 있다.
4. 대용량 분할: `ADMIN_MEMBER_EXPORT_PAGE_SIZE`를 초과하는 조건에서 여러 XLSX 파일이 생성되고 진행 카운트가 증가한다.
5. ZIP 묶음: 분할 파일이 ZIP으로 묶이고 ZIP 내부 파일을 열 수 있다.
6. 실패 메시지: 파일 생성 또는 ZIP 생성 실패 시 화면 progress에 실패 메시지가 표시되고 로그에 실패 원인이 남는다.
7. 임시 파일 정리: `adm/member_list_file_delete.php`에서 오늘 날짜 제외 임시 export 파일이 삭제된다.
8. 로그 기록: 성공/실패 모두 export 조건, actor, 결과가 export log 경로에 기록된다.

## 회원 인증과 가입

1. 로그인 성공/실패: 정상 계정은 로그인되고, 잘못된 비밀번호는 기존 오류 메시지로 종료된다.
2. 로그아웃: 세션이 정리되고 로그인 전 화면으로 돌아간다.
3. 회원가입 화면: 약관 동의, 본인확인 설정, 필수 입력값 검증이 기존 메시지로 동작한다.
4. 회원가입 저장: 신규 회원이 생성되고 이메일 인증 설정에 따라 인증 메일 또는 완료 화면이 표시된다.
5. 이메일 인증: 인증 링크 처리 후 `mb_email_certify`가 갱신되고 인증 토큰이 정리된다.
6. 이메일 변경 인증: 대기 중인 이메일이 기존 회원과 중복되지 않을 때만 새 이메일로 반영된다.
7. 이메일 수신중지: 정상 토큰일 때 수신중지 처리가 완료되고 비정상 토큰은 기존 오류로 종료된다.
8. 비밀번호 찾기/재설정: 인증 메일 발송, 인증 링크, 새 비밀번호 저장 흐름이 순서대로 완료된다.
9. 회원정보 수정: 기존 회원 정보가 저장되고 이메일 변경 또는 본인확인 관련 검증이 유지된다.
10. 회원탈퇴: 탈퇴 요청 후 회원 상태가 변경되고 로그인 세션이 정리된다.

## 관리자 회원 관리

1. 회원 목록: 검색, 정렬, 페이지 이동, 상태 필터가 기존 결과와 일치한다.
2. 일괄 수정: 선택 회원의 권한, 차단일, 메모 등 수정값이 저장된다.
3. 일괄 삭제: 권한이 부족하거나 자기 자신 삭제 같은 금지 케이스가 기존 메시지로 막힌다.
4. 회원 등록/수정: 필수값, 아이디 중복, 이메일 중복, 권한 검증이 유지된다.
5. 관리자 shell: page title, subtitle, container class가 `admin_apply_page_view()` 결과대로 반영된다.
